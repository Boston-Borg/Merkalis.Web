What Kastoria does

Trusted, analysis-ready data products.

Across healthcare and beyond, enterprise data is massively fragmented, multimodal, and untrusted. Kastoria sits below applications and above raw storage, turning that data into a cryptographically verifiable system of record.

Branch 1 — Ingest

Multimodal by design

Ingest and normalize diverse data types — imaging, signals, files, metadata, events — without format lock-in. EHR, PACS, genomics, claims, and vendor-proprietary data flow into a single repository.

Branch 2 — Govern

Policy-driven governance

Enforce governance across organizational and cloud boundaries. Fine-grained access control at the group, item, and projection level — so de-identified research views live alongside clinical data, safely.

Branch 3 — Verify

Cryptographically verifiable

Immutable by default, versioned, and forkable. Data integrity, lineage, and tamper resistance are verified by cryptographic hash — not reconciled after the fact.

Branch 4 — Replicate

Hybrid & multi-cloud, by default

Deploy on-prem, in a single cloud, across AWS, GCP, and Azure — or all at once. Kastoria keeps data in sync across regions and providers, with policy-controlled residency.

Branch 5 — Process

Customizable processing pipelines

Run WebAssembly-backed processing inside the repository, with configurable triggers and digital-signature policies for vendor updates. Compute co-locates with the data.

Branch 6 — Operationalize

AI-ready. Audit-ready.

Every data product exported from Kastoria carries its own proof — of source, transformations, and policy. Ready for regulated AI pipelines, longitudinal research, and clinical trials.

Product

Kastoria.

A multimodal data and AI infrastructure platform. Kastoria aggregates, secures, and governs enterprise data into a cryptographically verifiable system of record — across hybrid and multi-cloud environments.

Seven core capabilities

One foundation. Every trust property.

Kastoria is not a VNA, data lake, or object store replacement. It is a foundational trust layer, deployable anywhere — on-prem, cloud, edge, or hybrid.

1 · Flexible deployment
One or many clouds, on-prem, or edge Single cloud, one or more regions, or multi-cloud across AWS, GCP, and Azure. Supports on-premise datacenters alongside cloud deployments, with automatic sync. No RDBMS required; data lives in file systems and/or object storage.
2 · Customizable pipelines
WebAssembly processing, in-repository Flexible security policies govern access to outside resources. Configurable triggers execute processing logic on ingestion. Automatic deployment of vendor updates via digital-signature policies. GPU compute and external HTTP resources (FHIR, DICOMweb, any endpoint) are first-class.
3 · Automatic deduplication
Project many views, store once Different de-identified projections for research and AI workloads, performance-optimized views for different use cases — all backed by a single deduplicated pixel store. Drives cost reduction, data normalization, and discovery.
4 · Fine-grained access control
Groups. Items. Projections of items. Granular authorization down to individual data items, and across projections of those items. De-identified research data lives in the same repository as clinical data — global access mediated by policy, not physical separation.
5 · Immutable by default
Versioned, cryptographically verified, forkable All data is versioned and never destroyed unless explicitly deleted (e.g., medical-legal / risk reduction). Data, processing logic, mutation history, and logs all stored in the repository. Integrity verified by cryptographic hash. Repositories can be forked, branched, or snapshotted.
6 · Eventually consistent
CRDT-backed, offline-first All mutations done through CRDTs. Offline / disconnected operation first. Mutations are atomic — clients never see incomplete changes. Clients hold point-in-time snapshots and get notified when new versions arrive (e.g., a new image while a radiologist is reading a study).
7 · Cost effective
Fewer systems. Less hardware. Less overhead. Automatic deduplication and lossless compression reduce storage volume. Mix fast file systems, object stores, and deep archive on the same platform. Run compute on servers, VMs, containers, serverless, browser, or GPU — one storage backbone for all of it. Streamlined updates reduce management overhead.

About

We believe the integrity of a system should be a property of its structure — not a report you run after the fact.

Our thesis.

Enterprise data platforms were built for a world where storage was the bottleneck. Today, trust is. Regulators, AI systems, and clinical partners all need to agree on the same verifiable source of truth — and the current stack reconciles that after the fact, slowly, at extraordinary cost.

Merkalis inverts that. Integrity is constructed into every write, not audited out of every read. We build the structural intelligence layer that makes auditability a property of the data itself.

Our stage.

Early-stage. Kastoria is in active deployment with our first paying customer, MediQuant, as a forcing function for shippable, production-grade infrastructure.

Our H1 2026 focus is closing the gap between working technology and a general-availability product — in parallel with SOC 2 and HIPAA compliance tracks.
